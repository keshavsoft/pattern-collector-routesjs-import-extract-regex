Route JS Import Extract

A high-performance ESM import statement analyzer targeting aliased Express router imports from one-level relative folders.

🚀 Zero Dependencies 📂 Relative ./folder Focus 📦 ESM Native 🔥 Pure JS

📦 Installation

Install the package into your project using npm:

npm install pattern-collector-routesjs-import-extract

Or yarn:

yarn add pattern-collector-routesjs-import-extract

🛠️ Basic Usage

Import and invoke the default function to extract routes:

import routeImportExtract from 'pattern-collector-routesjs-import-extract';

const results = routeImportExtract({
  fileContent: codeText,
  inShowLog: false
});

🛠️ API Signature Reference

The default exported function accepts a single configuration object parameter:

routeImportExtract({ fileContent, inShowLog })
Parameter Type Description
fileContent string The raw JavaScript file content to scan.
inShowLog boolean Optionally log matches to the console (default: false).
⚠️ Parser Regex Contract: Current v7 uses /import\s*\{[^}]*router\s+as\s+(\w+)[^}]*\}\s*from\s*['"]\.\/([^/]+)\/.*['"]/. It captures variable from router as alias and folderName from the first folder after ./. It does not restrict the filename to routes.js.

🧭 How Matching Works

The package runs in two stages. First, pattern-collector-routesjs-import collects relative import lines. Then this package applies parseRegex to keep only named router alias imports from ./<folder>/....

const parseRegex = /import\s*\{[^}]*router\s+as\s+(\w+)[^}]*\}\s*from\s*['"]\.\/([^/]+)\/.*['"]/;

These match because both use router as alias and start with a one-level relative folder:

import { router as routerFromv1 } from "./v1/routes.js";
import { router as routerFromdoctors } from "./doctors/end-points.js";

These do not match:

import express from "express";
import routerFromv1 from "./v1/routes.js";
import { router } from "./v1/routes.js";
import { router as routerFromv1 } from "../v1/routes.js";

📋 Real-World Test Example (test/v3)

Here is the actual implementation of the test suite (test/v3) showing how the library extracts router imports from a routes.js file:

1. The Router File (test/v3/routes.js)

import express from 'express';

import { router as routerFromv1 } from "./v1/routes.js";
import { router as routerFromv2 } from "./v2/routes.js";
import { router as routerFromv3 } from "./v3/routes.js";
import { router as routerFromdoctors } from "./doctors/end-points.js";
import { router as routerFrommasters } from "./masters/routes.js";

const router = express.Router();
router.use("/masters", routerFrommasters);
router.use("/doctors", routerFromdoctors);
router.use("/v3", routerFromv3);
router.use("/v2", routerFromv2);
router.use("/v1", routerFromv1);

export { router };

2. The Test Script (test/v3/test.js)

import fs from 'fs';
import path from 'path';
import routeImportExtract from 'pattern-collector-routesjs-import-extract';

const filePath = path.join(process.cwd(), "routes.js");
const fileContent = fs.readFileSync(filePath, 'utf8');

const results = routeImportExtract({
    fileContent,
    inShowLog: false
});

console.log("ssssssssss : ", results);
